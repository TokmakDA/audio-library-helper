# Структура проекта

Краткая карта проекта для разработки и поддержки.

## Основные файлы и папки

- `manifest.json` — манифест расширения.
- `js/config.js` — общий конфиг, тексты интерфейса и значения по умолчанию.
- `js/content.js` — content script, связывает страницу, popup и service worker.
- `js/parser.js` — page script для Яндекс Музыки: следит за страницей, кнопками скачивания и сетевыми запросами.
- `js/service_worker.js` — логика скачивания, упаковка метаданных, очередь загрузок и runtime-сообщения.
- `js/popup.js` — интерфейс popup: текущий трек, следующий трек и действия по странице.
- `js/options.js` — страница параметров и рендер changelog.
- `html/popup.html`, `html/popup.css` — разметка и стили popup.
- `html/options.html`, `html/options.css` — разметка и стили страницы параметров.
- `data/changelog.json` — список изменений, который рендерится на странице параметров.
- `icons/` — иконки расширения.

## Поток данных

1. `content.js` внедряет `parser.js` в страницу Яндекс Музыки.
2. `parser.js` следит за запросами сервиса и пишет данные в `localStorage` страницы.
3. `content.js` синхронизирует это состояние в `chrome.storage.local`.
4. `service_worker.js` скачивает треки, получает метаданные и обновляет данные для popup.
5. `popup.js` читает состояние из `chrome.storage.local` и показывает текущий трек, следующий трек и действия для альбома, плейлиста или артиста.
